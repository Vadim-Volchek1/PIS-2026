# Use Case: Зарегистрировать баг

## Акторы
- Репортёр (пользователь)
- Сервис баг-трекера
- База данных
- Шина событий

## Happy path
1. Репортёр отправляет команду `CreateBug`.
2. Сервис валидирует входные данные (проект, приоритет, заголовок).
3. Сервис формирует агрегат `Bug` со статусом `OPEN`.
4. Агрегат сохраняется в БД в транзакции.
5. Публикуется событие `BugCreated`.
6. Пользователь получает `bug_id`.

## Error path
- Если проект не найден -> `404`.
- Если приоритет вне допустимого списка -> `400`.
- Если публикация события неуспешна -> событие пишется в outbox, ответ `202`.
