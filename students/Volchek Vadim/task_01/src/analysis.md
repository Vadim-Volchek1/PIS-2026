# Анализ транзакции

| Шаг | Синхронно | Риск отказа | Компенсация |
|---|---|---|---|
| Валидация команды | Да | Некорректные данные | Отказ без изменений |
| Сохранение бага | Да | Ошибка БД | Rollback транзакции |
| Публикация события | Нет (через outbox) | Недоступна шина | Retry воркером |

## Инварианты
1. Баг всегда принадлежит существующему проекту.
2. Приоритет только из набора `LOW`, `MEDIUM`, `HIGH`, `CRITICAL`.
3. Нельзя назначить исполнителя для закрытого бага.
