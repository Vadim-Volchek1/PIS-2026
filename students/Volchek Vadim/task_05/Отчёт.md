## Ход выполнения работы

### 1. Repository (PostgreSQL)

**Реализованные методы:**
- `save(request)` - сохранение агрегата бага
- `find_by_id(request_id)` - поиск бага по ID
- `find_active_requests()` - список багов в работе

**Технологии:** SQLAlchemy ORM + PostgreSQL.

---

### 2. REST Controller

**Эндпоинты:**

## Контрольные вопросы

1. **Почему Repository находится в Infrastructure, а не в Domain?**  
   Потому что Repository зависит от технических деталей хранения (ORM, SQL, транзакции), а Domain должен оставаться независимым от инфраструктуры.

2. **В чём преимущество ORM над обычным SQL?**  
   ORM ускоряет разработку, даёт объектную модель данных и снижает объём шаблонного SQL-кода, при этом сохраняя возможность тонкой настройки.
