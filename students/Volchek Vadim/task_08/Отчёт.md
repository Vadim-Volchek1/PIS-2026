## Цель работы

Разбить монолит баг-трекера на микросервисы с асинхронной коммуникацией через Event Bus.

---

## Вариант №11 - Баг-трекер «Не баг, а фича?»

---

## Ход выполнения работы

### 1. Request Service

**Bounded Context:** управление багами.

**API:**
- `POST /bugs`
- `GET /bugs/{id}`

---

### 2. Group Service

**Bounded Context:** управление командами разработки и назначением исполнителей.

**API:**
- `POST /teams`
- `GET /teams/{id}`

---

### 3. Event Bus (RabbitMQ)

**События:**
- `BugCreated`
- `BugAssigned`
- `BugActivated`
- `BugClosed`

---

### 4. API Gateway

**Маршрутизация:**
- `/bugs/**` -> Request Service
- `/teams/**` -> Group Service

**Конфигурация:** `src/api-gateway/nginx.conf`

## Вывод

Для баг-трекера реализована микросервисная схема с API Gateway и асинхронным обменом событиями через RabbitMQ.  
Выделение bounded contexts и применение circuit breaker повышают устойчивость системы и упрощают масштабирование.
